# 运维与可观测性

## 观测目标

- 快速定位问答失败、检索质量下降、任务堆积和外部依赖异常
- 支持容量评估、成本跟踪和 SLA 管理
- 为评测和优化提供稳定数据来源

## 日志

- API 访问日志：请求、响应码、耗时、租户、用户、Trace ID
- 任务日志：任务类型、输入对象、状态流转、错误信息
- 模型调用日志：Provider、模型、时延、令牌、错误码
- 检索日志：召回数量、过滤数量、重排前后 top_k、引用命中情况

## 指标

- 在线指标：QPS、P95/P99 延迟、错误率、超时率、拒答率
- 检索指标：召回条数、重排命中率、空结果率、引用覆盖率
- 任务指标：队列长度、吞吐、失败率、重试次数、平均处理时长
- 资源指标：CPU、内存、GPU、磁盘、索引容量、对象存储容量
- 业务指标：活跃租户数、知识库数量、文档增量、反馈满意度

## Tracing

- 使用 OpenTelemetry 贯穿网关、平台服务、检索服务、生成服务、Provider 调用
- `trace_id` 进入 API 响应和审计记录
- 关键 Span：鉴权、权限过滤、向量召回、全文召回、重排、生成、存储访问

## 告警

- API 错误率和延迟突增
- 任务队列堆积和消费停滞
- 模型调用超时或失败率过高
- 向量库、搜索引擎、数据库不可用
- 引用命中率或召回率持续下降

## SLA / SLO 建议

- 在线问答可用性 `>= 99.9%`
- P95 响应时间目标根据模型形态分级，例如 3 到 8 秒
- 文档导入成功率 `>= 99%`
- 高优先级索引任务完成时效按分钟级保障

## 运维动作

- 支持索引重建、任务重放、死信队列处理、配额调整
- 支持按租户限流、熔断和服务降级
- 出现模型不可用时切换到备选模型或仅返回检索结果摘要
