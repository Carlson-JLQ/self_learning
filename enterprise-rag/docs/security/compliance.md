# 安全与合规

## 安全目标

- 确保跨租户、跨知识库数据严格隔离
- 确保文档、索引、问答请求和配置变更全链路可审计
- 确保模型访问受控，避免敏感数据越权流出

## 认证与鉴权

- 用户认证支持 OIDC、SAML、LDAP 映射
- API 集成支持 API Token、OAuth2 Client Credentials、服务账户
- 所有接口经过统一鉴权中间件
- 权限校验顺序：租户状态 -> 用户身份 -> 角色权限 -> 数据作用域

## 数据隔离

- 每个业务实体必须带 `tenant_id`
- 检索前做数据权限过滤，禁止先召回后过滤
- 对高敏租户可选独立索引命名空间或独立存储实例
- 会话和审计记录按租户隔离，不支持跨租户检索

## 数据保护

- 原始文档、解析结果和导出文件默认加密存储
- 敏感字段支持脱敏展示，例如手机号、身份证号、合同金额
- Prompt 构造前可插入敏感信息清洗器
- 支持文档有效期和逻辑销毁策略

## 模型调用治理

- Provider 配置白名单，限制租户可用模型集合
- 对外部模型调用要求明确网络出口和审计策略
- 记录模型名称、版本、调用时间、令牌消耗、返回状态
- 对敏感租户默认关闭公网模型路由

## 审计要求

- 登录、权限变更、配置变更、文档导入、索引操作、问答请求、反馈提交全部记审计
- 审计事件不可被普通业务用户修改
- 审计日志至少支持按租户、操作者、资源、时间范围检索

## 合规建议

- 对接企业数据分级分类制度，给文档和知识库打密级标签
- 明确数据留存周期、访问审批流程、导出审批规则
- 对接企业 SIEM、日志平台或堡垒机审计体系
- 在正式上线前完成渗透测试、权限测试、敏感数据流向评估
