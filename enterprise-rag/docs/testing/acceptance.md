# 测试与验收

## 测试策略

- 单元测试覆盖领域规则、权限校验、Provider 抽象适配
- 集成测试覆盖文档处理、索引构建、检索、生成与审计闭环
- 端到端测试覆盖控制台操作、API 接入和典型问答链路
- 性能测试覆盖批量导入、高并发问答、索引重建

## 核心验收场景

### 文档接入

- 支持 PDF、Word、Markdown、HTML、TXT 上传和状态追踪
- 文档损坏、密码保护、格式不支持时返回明确错误码
- 重复上传相同文件可识别并按策略跳过或生成新版本

### 索引与检索

- 文档成功生成切片、向量和全文索引
- 权限过滤不会泄露无权文档
- 混合检索相比单一召回具备更高命中率
- 索引重建期间旧版本可持续提供查询

### 问答与引用

- 回答必须包含至少一个有效引用，或明确拒答原因
- 多轮会话中历史上下文受长度约束并可正确裁剪
- 对知识库外问题应拒答，不生成无依据内容

### 安全与审计

- 跨租户访问被拒绝并记录审计
- 角色变更、知识库授权变更、模型路由变更均可审计
- 敏感文档在无权限时不会进入召回候选

### 稳定性

- 模型超时、向量库故障、搜索引擎不可用时具备降级和重试
- Worker 异常退出后任务可恢复
- 大批量导入下队列不出现无限积压

## 质量指标

- 召回率、MRR、NDCG
- 答案正确率、引用命中率、拒答准确率
- 首字节时间、完整响应时间
- 用户反馈满意度、人工复核通过率

## 测试数据建议

- 建立标准问答集，覆盖制度、合同、手册、FAQ、技术文档
- 为每个问题标注标准答案、目标文档、关键引用片段
- 对高敏数据准备脱敏测试集和权限隔离测试集
