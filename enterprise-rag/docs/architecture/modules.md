# 模块设计

## 1. 控制台与开放接入

### 管理控制台

- 租户、用户、角色、知识库、模型、索引任务、审计、告警配置管理
- 面向管理员、运营人员、知识库维护人员
- 与后端仅通过受控 API 通信，不直连数据层

### 开放 API

- 面向业务系统和外部应用集成
- 提供同步查询接口和异步任务接口
- 强制接入鉴权、限流、审计

## 2. 租户与权限模块

- 支持租户级资源隔离和配额管理
- RBAC 定义到资源动作级别，数据权限进一步约束知识库与文档访问范围
- 默认角色包括 `tenant_admin`、`kb_admin`、`kb_editor`、`kb_reader`、`auditor`
- 所有检索与问答请求都必须先做权限过滤，再做召回

## 3. 知识库与文档模块

- 知识库是逻辑隔离单元，绑定检索策略、模型路由、可访问用户组
- 文档支持版本、标签、来源系统、语言、密级、有效期等元数据
- 接入方式包括文件上传、对象存储导入、Web 抓取、API 推送、连接器扩展
- 文档状态机建议为 `uploaded -> parsing -> chunking -> indexing -> active / failed`

## 4. 文档处理管线

- Parser：统一产出结构化段落、标题、表格、附件引用
- Cleaner：去噪、去页眉页脚、统一编码、特殊字符清洗
- Chunker：按标题层级、段落长度、语义边界切片，保留父子关系
- Embedder：批量生成向量，支持模型版本号和重复计算去重
- Indexer：向量索引、全文索引、元数据索引一体构建

设计要求：

- 每一步输出独立状态和可重试错误码
- 管线支持断点续跑和局部重建
- 处理结果可审计，可回溯至原始文档版本

## 5. 检索与生成模块

### 检索服务

- 输入：问题、知识库范围、用户上下文、过滤条件
- 输出：候选片段、相关性分数、召回来源
- 召回顺序：权限过滤 -> 倒排召回 -> 向量召回 -> 合并去重 -> 重排

### 重排服务

- 独立服务化，便于替换本地模型或外部服务
- 只处理已过滤候选集，避免高成本全量重排

### 生成服务

- 负责 Prompt 组装、上下文压缩、模型调用、引用整理
- 拒答策略包括无足够证据、权限不足、问题超出知识域
- 响应必须附带引用、模型信息、耗时和追踪 ID

## 6. 评测与反馈模块

- 支持人工反馈：有帮助、无帮助、引用错误、事实错误
- 支持离线评测数据集：问题、标准答案、候选文档、预期引用
- 指标包括召回率、MRR、答案正确率、引用命中率、拒答准确率
- 评测结果用于调整切片、召回参数、重排模型和 Prompt 模板

## 7. 审计与运营模块

- 记录登录、配置变更、知识库授权变更、文档导入、问答请求、模型调用
- 对敏感操作保留操作者、时间、租户、资源、结果、客户端来源
- 运营视图聚焦租户活跃度、知识库命中率、失败任务、调用成本和时延

## 8. 配置与扩展机制

- Provider 配置在租户级和全局级均可覆盖
- 检索策略支持按知识库配置：`top_k`、混合权重、重排阈值、拒答阈值
- 连接器与 Provider 按插件方式注册，避免主干代码分叉
